# GME Test Agent 源码运行使用说明

本文档面向从 GitHub 克隆 GME Test Agent 源码并在 Windows 本机运行的使用者。源码模式不要求固定名称的 Conda 环境，可以使用系统 Python、Conda、`.venv` 或其他虚拟环境。

## 1. 环境要求

### 1.1 操作系统

- Windows 10/11 x64
- PowerShell 5.1 或更高版本

### 1.2 Python

- Python 3.10 或更高版本
- 推荐 Python 3.11
- 当前环境需要能够执行 `python`，或者在脚本中通过 `-Python` 指定解释器路径

环境名称没有限制。例如，以下环境均可使用：

```text
系统 Python
Conda 环境
.venv
virtualenv
```

### 1.3 Node.js

- Node.js 18 或更高版本
- 推荐 Node.js 20 或 22 LTS
- npm 需要在 `PATH` 中

Node.js 仅用于从源码构建和启动 Vue 前端。使用 Release 中的桌面 EXE 时不需要自行安装 Node.js。

### 1.4 GME 开发工具

本机需要安装并配置：

- 完整的 GME 仓库
- Git
- CMake
- Visual Studio 2022 C++ Build Tools
- MSVC v143 C++ 编译工具
- Windows 10/11 SDK
- `clang-format`
- GitHub CLI `gh`

建议将 `git`、`cmake`、`clang-format` 和 `gh` 加入 `PATH`。

### 1.5 Codex 与 GitHub 登录

- 当前 Windows 用户需要具有可用的 Codex 登录状态
- `C:\Users\<用户名>\.codex\auth.json` 应存在，或者已设置 `OPENAI_API_KEY`
- GitHub CLI 需要完成登录，并具有目标测试仓库的 push 和 PR 权限

检查 GitHub 登录：

```powershell
gh auth status
```

## 2. 获取源码

```powershell
git clone https://github.com/maincourses/gme-test-agent.git
cd gme-test-agent
```

## 3. 选择 Python 环境

可以直接使用当前 Python，也可以先激活任意虚拟环境。

使用 `.venv`：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

使用任意 Conda 环境：

```powershell
conda activate <环境名称>
```

确认当前解释器：

```powershell
python --version
python -c "import sys; print(sys.executable)"
```

## 4. 一键初始化源码环境

在仓库根目录运行：

```powershell
scripts\setup_source.ps1
```

初始化脚本会：

1. 使用当前终端中的 Python。
2. 检查 Python 是否为 3.10 或更高版本。
3. 安装 `requirements.txt` 中的 Python 依赖。
4. 检查 Node.js 和 npm。
5. 在 `frontend` 目录执行 `npm ci`。
6. 缺少 `config.local.json` 时，根据模板自动创建。
7. 检查 Git、CMake、Visual Studio 2022 C++ Build Tools、`clang-format`、`gh` 和 Codex 登录状态。

指定 Python 解释器：

```powershell
scripts\setup_source.ps1 -Python "C:\Path\To\python.exe"
```

只初始化后端、不安装前端依赖：

```powershell
scripts\setup_source.ps1 -SkipFrontend
```

## 5. 手动安装依赖

不使用初始化脚本时，可以手动执行：

```powershell
python -m pip install -r requirements.txt

cd frontend
npm ci
cd ..

Copy-Item config.example.json config.local.json
```

`requirements.txt` 只管理 Python 依赖；前端依赖由 `frontend/package-lock.json` 和 `npm ci` 管理。

## 6. 配置 config.local.json

`config.local.json` 是每台电脑独立的本地配置，已被 `.gitignore` 排除，不会提交到 GitHub。

初始化脚本会自动把以下路径设置到当前项目目录：

```text
worktrees\
artifacts\
gme_agent.db
```

使用者必须修改：

```json
"gme_repo_path": "D:/本机的GME仓库路径"
```

同时检查：

```json
"base_branch": "main",
"github_remote": "origin",
"test_target_repo": "tests/gme",
"module_repo_root": "module"
```

路径建议使用正斜杠 `/`，或者对 Windows 反斜杠进行转义。

## 7. 启动应用

确保当前终端已经激活安装过依赖的 Python 环境，然后运行：

```powershell
scripts\run_web.ps1
```

脚本会：

1. 解析当前终端的 `python` 绝对路径。
2. 检查 Python 版本和 `openai-codex` 依赖。
3. 检查 `config.local.json`、Node.js 和 npm。
4. 在本机回环地址启动带随机 API Token 的后端。
5. 启动 Vite 前端。

默认地址：

```text
前端：http://127.0.0.1:5173
后端：http://127.0.0.1:8765
```

指定其他 Python：

```powershell
scripts\run_web.ps1 -Python "C:\Path\To\python.exe"
```

为兼容旧用法，仍可显式指定 Conda 环境：

```powershell
scripts\run_web.ps1 -CondaEnv <环境名称>
```

`-CondaEnv` 不再有默认值，也不再强制环境名称必须是 `py311`。

## 8. 首次运行

应用启动后：

1. 打开“设置”页面。
2. 检查 GME 仓库、worktree、artifact 和数据库路径。
3. 点击“环境检查”。
4. 所有必要检查通过后再创建测试任务。

`gme_agent.db` 不需要手动创建，后端会在首次启动时自动初始化。

## 9. 运行项目测试

使用当前 Python 环境：

```powershell
scripts\run_tests.ps1
```

指定 Python：

```powershell
scripts\run_tests.ps1 -Python "C:\Path\To\python.exe"
```

显式使用任意 Conda 环境：

```powershell
scripts\run_tests.ps1 -CondaEnv <环境名称>
```

前端构建检查：

```powershell
cd frontend
npm run build
```

## 10. 打包桌面版

使用当前 Python 环境打包：

```powershell
scripts\package_desktop.ps1
```

指定 Python：

```powershell
scripts\package_desktop.ps1 -Python "C:\Path\To\python.exe"
```

显式使用 Conda 环境：

```powershell
scripts\package_desktop.ps1 -CondaEnv <环境名称>
```

生成文件位于：

```text
frontend\release\GME Test Agent-<版本>-x64.exe
```

## 11. 常见问题

### 找不到 Python

确认已经安装 Python，并且当前终端可以运行：

```powershell
python --version
```

也可以使用 `-Python` 传入完整路径。

### Python dependencies are missing

当前启动所使用的 Python 环境没有安装项目依赖：

```powershell
python -m pip install -r requirements.txt
```

### 找不到 Node.js 或 npm

安装 Node.js 18 或更高版本，重新打开终端并检查：

```powershell
node --version
npm --version
```

### 缺少 config.local.json

运行：

```powershell
scripts\setup_source.ps1
```

或者手动执行：

```powershell
Copy-Item config.example.json config.local.json
```

### Codex auth 检查失败

打开 Codex App 或 Codex CLI，使用当前 Windows 用户登录，然后重新执行环境检查。

### PowerShell 不允许运行脚本

可以在当前进程临时放开：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
```

### 构建 GME 测试失败

检查 Visual Studio 2022 C++ Build Tools、MSVC v143、Windows SDK、CMake、GME 仓库及其依赖是否完整，并在应用中重新执行环境检查。
